Parametrische Klassifikation und Naive Bayes Methode

Klassifikatoren mssen zu einer gegebenen Eingabe xx_ die zugehrige Klasse CiCi bestimmen. Mithilfe der Wahrscheinlichkeitstheorie kann diese Aufgabe wie folgt beschrieben werden: Bestimme fr alle mglichen Klassen CiCi die bedingte Wahrscheinlichkeit P(Ci|x)P(Ci|x_), also die Wahrscheinlichkeit, dass die gegebene Eingabe xx_ in Klasse CiCi fllt. Die Entscheidungsregeln von Klassifikatoren knnen mit Methoden des ""berwachten Lernens"" aus Trainingsdaten ermittelt werden. Im Fall des parametrischen Lernens kann aus den Trainingsdaten die sogenannte Likelihood-Funktion p(xCi)p(x_Ci) bestimmt werden. Anmerkung: Allgemein werden mit p(...)p(...) kontinuierliche Wahrscheinlichkeitsfunktionen und mit P(...)P(...) diskrete Wahrscheinlichkeitswerte bezeichnet.

Mithilfe der Bayes-Formel

P(Cix)=p(xCi)P(Ci)p(x)P(Cix_)=p(x_Ci)P(Ci)p(x_)

kann aus der Likelihood die a-posteriori-Wahrscheinlichkeit P(Cix)P(Cix_) berechnet werden. Darin wird P(Ci)P(Ci) die a-priori-Wahrscheinlichkeit und p(x)p(x_) die Evidenz genannt. Die a-priori-Wahrscheinlichkeit kann ebenfalls aus den Trainingsdaten ermittelt werden. Die Evidenz ist fr die Klassifikationsentscheidung nicht relevant, da sie fr alle Klassen CiCi gleich gro ist.

Die Berechnung der Likelihood-Funktion p(xCi)p(x_Ci) ist dann sehr aufwendig, wenn x=(x1,x2,,xZ)x_=(x1,x2,,xZ) ein Vektor von voneinander abhngigen Variablen xixi ist. Bei der Naive Bayes Classification wird jedoch von der vereinfachenden Annahme ausgegangen, dass die Eingabevariabeln xixi voneinander unabhngig sind. Dann vereinfacht sich die bedingte Verbundwahrscheinlichkeits-Funktion p(x1,x2,,xZCi)p(x1,x2,,xZCi) zu:

p(x1,x2,,xZCi)=j=1Zp(xj|Ci)p(x1,x2,,xZCi)=j=1Zp(xj|Ci)

class Classifier:

    """
    - ein dictionary mit instanzvariable fc und cc initialisiert
    - fc ist ein verschachteltes dictionary, 
      keys sind die bisher gelernten Worte, values zhlen wie hufig Worte vorkamen
    - getfeatures wird die Funktion getwords() bergeben und referenziert damit diese
    """
    def __init__(self, getfeatures, categories):
        self.fc = defaultdict(lambda: defaultdict(int))
        self.cc = defaultdict(int)
        self.getfeatures = getfeatures
        self.categorie = categories 
    
    """
    - wird ein Wort f bergeben
    - wird eine zugehrige Kategorie cat bergeben
    - der fc Zhler wird angepasst
    """
    def incf(self, f, cat):
        if(f in self.fc):
            self.fc[f][cat] = self.fc[f][cat]+1
        else:
            tempDict = {}
            for x in self.categorie:
                tempDict[x] = 0
            self.fc[f] = tempDict
            self.fc[f][cat]=self.fc[f][cat]+1
    
    """
    - wird ein Wort und die zugehrige Kategorie cat bergeben
    - hierber wird der klassen-eigene cc Zhler angepasst
    """
    def incc(self, cat):
        self.cc[cat] = self.cc[cat]+1
    
    """
    - gibt Hufigkeit des wortes f der Kategroie cat zurck
    """
    def fcount(self, f, cat):
        if(f in self.fc):
            return self.fc[f][cat]
        else:
            return 0
    """
    - gibt die Anzahl der Dokumente der Kategorie cat zurck
    """
    def catcount(self, cat):
        return self.cc[cat]
    
    """
    - gibt Anzahl aller Dokumente zurck
    """
    def totalcount(self):
        return sum(self.cc.values())

    """
    - wird ein neues Trainingselement mit
      Betreffzeile (item) und Kategorisierung (cat) bergeben
    - der String item wird mit der Instanzmethode getfeatures in Worte zerlegt.
    - Aufruf von incf(f, cat) je feature/Wort
    """
    def train(self, item, cat):
        features = self.getfeatures(item)
        for f in features:
            self.incf(f, cat)
        self.incc(cat)    

    """
    - berechnet die bedingte Wahrscheinlichkeit  P(f|cat)  des Wortes f in der Kategorie cat
    - je geringer die Anzahl an Worten, desto extremer das ergebnis
    - als ausgleich hierfr wird weightedprob implementiert
    """
    def fprob(self, f, cat):
        return self.fcount(f, cat)/self.catcount(cat)

    """
    - berechnet einen wahrscheinlichkeitswert
    - initProb ist ein initialier Wahrscheinlichkeitswert
    - je hufiger das wort vorkommt, desto mehr nimmt 
    """
    def weightedprob(self, f, cat):
        initProb = 0.5
        if(f in self.fc):
            count = sum(self.fc[f].values())
        else:
            count = 0
        return (initProb + count * self.fprob(f,cat))/(1+count)

    """
    - bergabe beliebigen Textdokumentes
    - Berechnung der bedingten Wahrscheinlichkeit mit
      a-postpriori-Wahrscheinlichkeit P(C/X) mit dem das neue Dokument X
      in die Kategorie y fllt
    """
    def prob(self, item, cat):
        features = self.getfeatures(item)
        produkt = 1
        for f in features:
            produkt = produkt * self.weightedprob(f, cat)
        return produkt * (self.catcount(cat)/self.totalcount())    
    
    """
    - Kategorisierung des dokumentes
    """
    def categorise(self, item, catA, catB):
        a = self.prob(item, catA)
        b = self.prob(item, catB)
        if a>b:
            return catA
        else:
            return catB

def precision(tp, fp):
    return tp/(tp+fp)
    
def recall(tp, fn):
    return tp/(tp+fn)

def accuracy(tp, tn, total):
    return (tp + tn)/total

def f1_score(precision, recall):
    return 2*((precision*recall)/(precision+recall))

Die Confusion Matrix ist eine Methode zur Leistungsmessung fr Klassifizierungsproblematiken des maschinellen Lernens. Die Ausgabe (engl., Output) kann zwischen einer und mehrerer Klassen liegen. Visuell dargestellt ist die Confusion Matrix eine Tabelle mit 4 verschiedenen Kombinationen (TP, FP, FN, TN) zwischen vorhergesagten und tatschlichen Werten.

Recall

Die Mazahl des Recall (dt., Sensitivitt) oder Richtig-Positiv-Rate des Klassifikators erlaubt Aussagen darber, wie viele Werte korrekt vorhergesagt wurden.

Precision

Die Mazahl der Precision (dt., Genauigkeit) ermglicht eine Aussage darber, wie viele positiv vorhergesagten Werte tatschlich der positiven Klasse zugeordnet werden knnen.

Accuracy

Die Mazahl der Accuracy erlaubt eine Aussage ber den Anteil der korrekt klassifizierten Flle. Bei unausgeglichenen (engl., biased) Testdaten wird mit der Accuracy die Effektivitt eines Klassifikators nicht mehr erfassbar.

F1-Score

Die Mazahl des F1-Score ist das harmonische Mittel aus Precision und Recall und wird hufig als zusammenfassende Metrik verwendet. Der F1-Score oder F-Ma ist daher ein guter Indikator fr die Genauigkeit der erzielten Ergebnisse von Recall und Precision.

def getwordsimproved(doc):
    toRet = {}
    #stopwort-Entfernung
    stop_words = set(stopwords.words('german'))

    tokens = tokenizer.tokenize(doc.lower())
   
    #Implementierung von Bigram
    ngram = True  #atm nicht in nutzung
    if ngram:
        docstring = ' '.join(tokens).strip()
        n = 4
        n_grams = ngrams(docstring.split(), n)
        tokens = tokens + [' '.join(grams) for grams in n_grams]
    
    for token in tokens:
        if not token in stop_words:
            if not token in toRet:
                toRet.update({token : 1})
            else:
                toRet[token] += 1 #Wenn Wort bereits in Dict enthalten +=1
    return toRet
